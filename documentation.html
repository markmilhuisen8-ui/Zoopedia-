🔑 jQuery Criteria Glossary

What each criteria means — explained so anyone can understand it, even without jQuery knowledge.

Criteria 1
DOM Traversal
Moving around the page structure — finding parent elements, child elements, or siblings of a selected element.
$(this).find('.card') → Search inside this element
Criteria 2
DOM Modification
Adding or removing things from the page at runtime — creating new HTML elements on the fly and injecting them, or deleting them.
$('.list').append('<li>New Item</li>')
Criteria 3
DOM Attribute Modification
Changing the properties or settings of an HTML element — like changing an image source, disabling a button, or setting custom data values.
$('img').attr('src', 'new.jpg')
Criteria 4
CSS Manipulation
Changing the visual style directly through code — changing colors, sizes, toggling class names that apply stylesheet rules to elements.
$('body').addClass('dark-mode')
Criteria 5
JavaScript Effects
Animated, timed visual transitions — like making something fade in/out smoothly, or slide a panel up or down over a specified duration.
$('#popup').fadeIn(300)
Criteria 6
HTML Events
Responding to user actions — like clicks, key presses, typing in a text box, or hovering over an element. These are the triggers for interactivity.
$('#btn').on('click', function() {})

Grid System, TV Signal Lost Effect & Search Lock

📁 main.js  |  📄 vaultpage.html

Purpose The Vault page shows all animals as a grid of retro-styled TV monitor cards. As the user types in the search box, non-matching animals get a "TV static corruption" visual effect and their "View" button changes to "LOCKED" — creating an immersive, branded filtering experience.
6HTML Events — Live Search Trigger & Hover Glow

How: .on('input') fires on every keystroke, .on('mouseenter/.mouseleave') handles card hover glow.

// Every time user types in search box (main.js Line 53)
  $('#animalSearch').on('input', function() {
      let value = $(this).val().toLowerCase().trim();
      // ...filter logic runs here for every key press
  });

  // Hover glow on card enter/leave (main.js Line 75)
  $(document).on('mouseenter', '.custom-card-tv', function() {
      $(this).css({ 'border': '2px solid #7711a7', 'box-shadow': '0 0 15px #ce9ee5' });
  });
1DOM Traversal — Find Elements Inside Each Card

How: .find() searches downward inside each wrapper div to reach the card and button elements.

// main.js Lines 60–61
  let $card = $(this).find('.custom-card-tv'); // Go into this wrapper, find the card
  let $btn  = $(this).find('a');              // Go into this wrapper, find the link button
4CSS Manipulation — Apply & Remove Signal Lost Style

How: .addClass('signal-lost') triggers the TV static animation. .css('pointer-events','none') makes "LOCKED" buttons un-clickable.

// main.js Lines 64–68
  if (name.includes(value)) {
      $card.removeClass('signal-lost is-locked'); // Card matches: restore normal
      $btn.css('pointer-events', 'auto').text('View');
  } else {
      $card.addClass('signal-lost is-locked');    // No match: apply glitch effect
      $btn.css('pointer-events', 'none').text('LOCKED');
  }
2DOM Modification — Build the Entire Card Grid at Runtime

How: Animal cards are not hard-coded in HTML. .append(cardHtml) dynamically generates and injects every card from the animal data.

// main.js Lines 26–42
  let cardHtml = `<div class="animal-wrapper" data-name="${animal.animalname}">
      <div class="custom-card-tv">
          <div class="static-overlay"></div>
          <img src="${animal.image}">
          <h5>${animal.animalname}</h5>
      </div>
  </div>`;
  container.append(cardHtml); // Injects each card into the grid

Dark Mode & Light Mode

📁 darkmode.js  |  📄 All pages (navbar)

Purpose A BB8 Droid-themed toggle switch in the navbar switches the entire website between a dark space-themed aesthetic and a bright light mode. The choice is saved in localStorage and remembered across every page and after browser restarts.
4CSS Manipulation — One Class Changes Everything

How: Adding 'dark-mode' to <body> triggers hundreds of CSS rules simultaneously. All dark-mode colors and backgrounds are defined in style.css under body.dark-mode.

// darkmode.js Lines 9 & 13
  $body.addClass('dark-mode');     // → Dark: adds class, triggers all dark CSS rules
  $body.removeClass('dark-mode');  // → Light: removes class, reverts all styles
3DOM Attribute Modification — Syncing the Toggle Switch Visually

How: .prop('checked') sets the checkbox's checked state so the BB8 ball moves to the correct position when the theme is loaded from storage.

// darkmode.js Lines 11 & 15
  $checkbox.prop('checked', true);  // Dark mode → move BB8 to the right
  $checkbox.prop('checked', false); // Light mode → move BB8 to the left
6HTML Events — The Toggle Switch Listener

How: Listens for change and click events on the BB8 checkbox. .is(':checked') reads the current state to determine which theme to apply.

// darkmode.js Lines 23–25
  $(document).on('change click', '.bb8-toggle__checkbox', function() {
      setTimeout(() => updateTheme($(this).is(':checked')), 10);
  });

Contact Form Validation & Mr. Bean Popup

📁 validatonsignal.js  |  📄 contactpage.html

Purpose As the user types their email, real-time feedback appears: green tick (valid), yellow warning (missing @), red cross (invalid). On submit, a custom modal popup appears with Mr. Bean celebrating or showing disappointment depending on the validation state.
6HTML Events — 3 Triggers
// TRIGGER 1: Real-time validation as user types (Line 2)
  $('#email-input').on('input', function() {
      const val = $(this).val();
      // Validation runs here on EVERY keystroke
  });

  // TRIGGER 2: Submit button click → show popup (Line 71)
  $('#submit-btn').on('click', function(e) {
      e.preventDefault(); // CRITICAL: prevents page reload
  });

  // TRIGGER 3: Close button click → dismiss popup (Line 92)
  $('.close-btn').on('click', function() {
      $('#modal-container').fadeOut(300);
  });
5JavaScript Effects — Smooth Icon & Popup Transitions

AI Assistant + Magic Hand Gesture Control

📁 handtracking.js, NomadAi.js, mediacontrols.js  |  📄 All pages

Purpose A dual touchless interface. Magic Hand: webcam tracks the user's hand to move a custom cursor, click elements (middle+thumb pinch) and scroll (index+thumb pinch). Nomad AI: clap to activate, then speak commands in English or Sinhala to navigate, switch themes, or control media. Media Controls: two floating buttons on every page toggle camera and microphone hardware completely — state persists across navigation.
Advanced AI Logic Upgrades (New) • Conversational Intent Filter: Prevents accidental navigation by detecting conversational phrases (e.g., "can you tell me") vs. direct commands. • Cross-Page State Tracking: Uses sessionStorage to allow Alex to continue voice scripts after navigating (used for the "Dev Team" intro). • Phonetic Name Correction: Direct TTS engine override ensures Sri Lankan names like Ameendree and Minasi are pronounced perfectly while text captions remain pristine. • Markdown Sanitization: Automatically strips symbols like # and * before speech to prevent the engine from reading out symbols (e.g., "hashtag hashtag").
2DOM Modification — Custom Cursor & Control Panel Created by jQuery
// handtracking.js Lines 17–33: Cursor built entirely by jQuery

  $cursor.css({ position: 'fixed', clipPath: 'polygon(0 0, 0 85%, 28% 65%...)' });
  $('body').append($cursor); // Injects cursor — not in any HTML file

  // mediacontrols.js Lines 9–19: Floating panel built by jQuery
  const $panel = $(`<div id="zp-media-controls">
      <button id="zp-cam-btn"></button>
      <button id="zp-mic-btn"></button>
  </div>`);
  $('body').append($panel); // Injects on EVERY page automatically

      // e.originalEvent.isTrusted = true ONLY for real mouse clicks
      // Gesture-synthesized clicks have isTrusted = false → REJECTED

      // Only reaches here for genuine hardware clicks

      else { powerUp(); }
  });
